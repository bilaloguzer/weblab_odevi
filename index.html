Frontend - Backend

Frontend

WEB geliştirme temel olarak ikiye ayrılır: frontend ve backend olmak üzere. Frontend tarafı kullanıcının yazılan web sayfası/uygulamasıyla etkileşime geçtiği kısımdır. HTML, CSS, JavaScript kullanılarak yazılır. UI (user interface) ve UX (user exprerience) gibi alanları içine alır. Kullanıcının gördüğü ve etkilişime geçtiği kısım frontend'dir.

Backend

Frontend'de yer alan bu öğelere eklenen bilgilerin depolanabilmesi, yani kısaca frontend'in hayata geçebilmesi için gereken alt yapı ve teknolojiyi sağlayan ise backend'dir. Backend genellikle bir sunucu, bir uygulama ve bir veri tabanından oluşur. Frontend'in aksine çok daha fazla seçenek vardır backend'de çalışırken: JavaScript, Pyhton, Ruby, PHP, SQL...

Frontend ile Backend nasıl iletişim kuruyor ?

Frontend ve Backend'in kabaca ne olduğunu biliyorsunuz artık. Ama bu iki taraf birbiriyle nasıl iletişime geçiyor ? Hatta belki "neden iletişime geçiyor ?" bile diyebilirsiniz. Şöyle bir örnek vereyim: bir uçak bileti alacaksınız ve her bir değişkeni değiştirdikçe sizin karşınıza farklı rotalar, fiyatlar çıkıyor değil mi ? İnternet sitesi size hangi bileti göstereceğini nasıl biliyor ? Bütün uçuş bilgilerini sizin tarayıcınızda saklayamaz sonuçta, bu çok gereksiz ve verimsiz olurdu. Onun yerine sizin frontend'de oluşturduğunuz bir istek backend'e ulaşıp size geri dönüyor. Peki bu nasıl gerçekleşiyor ? Burada da API (Application Programming Interface) dediğimiz kavram karşımıza çıkıyor. Siz sunucuda saklanan bir bilgiyi görmek istediğinizde API sunucuya bir http talebi yolluyor ve sizin istediğiniz uçuş bilgisini size aktarıyor.

JavaScript

JavaScript Nedir ?

JavaScript, yaygın olarak web tarayıcılarında kullanılmakta olan bir programlama dilidir. JavaScript ile tarayıcının kullanıcıya etkileşimde bulunması, tarayıcının kontrol edilmesi, senkron bir şekilde sunucu ile iletişime geçilmesi ve web sayfası içeriğinin değiştirilmesi gibi işlevler sağlanır. 2022 verilerine göre internetin "client" tarafının 98%'inde JavaScript kullanılıyor. JavaScript olmasa web sitelerine dinamizm katamazdınız.

JS Frameworkleri

ANGULAR

Angular özünde TypeScript temelli bir geliştirici platformudur. Açık kaynak kodlu dinamik web uygulamaları için yazılmış yapısal bir framework'tür. 2021 Aralık'a kadar Google destekli bir framework'tü. Angular'ın özellikleri :

  • Geliştiriciye daha az kod ile daha fazla işlevsellik sunar.
  • Cross tarayıcı uyumludur.
  • Büyük bir ölçekte, verimli bir web uygulamasını bakımı kolay bir şekilde yazmanızı sağlar.

NODE JS

2009 yılında geliştirilmeye başlanmış, açık kaynaklı, "server side" uygulamalar yazılabilen bir JavaScript Runtime platformudur. Yani anlayacağınız backend tarafında kullanılıyor. Node.js'in bu kadar yaygın olmasında bir sürü sebep var. Bunlardan biri çok hızlı ve her ölçeğe uyarlanabilmesidir. Backend tarafında kullanılan dillerle kıyaslandığında aynı işi node ile daha az kodla daha kısa sürede yapıldığı gözlemlenmiş. Aynı zamanda JavaScript framework'ü olduğu için bir web geliştirici için öğrenmesi ve kullanması çok daha rahat.

REACT

2013 yılında Facebook'ta geliştirilen React Web'deki açık ara en popüler kütüphane. Kullanıcı arayüzleri yazmak için kullanılan bir kütüphanedir. Html ve JavaScript kodunu bir arada yazdığın JSX isimli bir JavaScript uzantısı kullanır. Kolaylığının yanı sıra popülerliğindeki en büyük diğer etken ise kütüphaneyi çevreleyen inanılmaz ekosistemi. Community'deki "GATSBY", "NEXT", "FORMIK", "REDUX", "MOBX" ve sayamadığım daha nice açık kaynak kütüphaneleri ile geliştiricilerin hayatını çok kolaylaştırıyor.

Genel Kavramlar

http://

Açılımı "Hyper Text Transfer Protocol" olan bu kavram kısaca bilginin sunucudan kullanıcıya nasıl aktarılacağını gösteren protokoldür. Tarayıcınızın yardımı ile bir web sayfasına girmek istediğiniz zaman http ile sunucuya bir istek gönderilir, sunucu isteğe cevap verdiği vakit sayfaya girmiş olursunuz. Eğer sunucudan cevap gelmezse hata alırsınız ve sayfaya giremezsiniz.

CORS

Açılımı "Cross-Origin Resource Sharing" (Kökenler arası kaynak paylaşımı) olan web sayfalarının birbirleriyle kaynak paylaşımı yaparken kontrolü sağlayan bir mekanizmadır. Kaynak dediğimizde kastedilen şeyler: medya(görsel,ses), css dosyası, js dosyası, fontlar vb... normalde zararsız gözüken bu sistem arka planda kaosa yol açabilir.

DOM

Açılımı "Document Object Model" olan DOM dökümanlara erişmek için kullanılan bir W3C standardıdır. DOM dokümandaki nesnelere erişmek ve içeriğini, stilini, yapısını değiştirmek için kullanılır. Yapısı bir ağaca benzeyen DOM kökünde HTML etiketi onun altında ise içersine yazılmış diğer etiketler bulunur. HTML DOM'a JavaScript ile erişim sağlayabilirsiniz.

JWT

Açılımı "JSON Web Token" olan RFC 7519'de tanımlanmış açık bir standarttır. Kullanıcı doğrulama, web servis güvenliği, bilgi güvenliği gibi konularda kullanılan oldukça popüler bir yöntemdir. Boyutunun küçük olması sayesinde URL üzerinden gönderilebilir, HTTP başlığı içerisinde yer alabilir ve küçük boyutu sayesinde aktarımı da hızlı gerçekleşir.

Cookies

HTML5'den önce "çerezler" kullanıcıların oturumlarıyla ilgili veri toplamak, kişiselleştirmek ve kaydetmek amacıyla web tarayıcıları için yapılmıştır. Çerezlerde depolanan veriler siz sunucudan bir şey istedikçe sunucuyla değiş-tokuş edilir buna göre de sunucu size özel olarak hangi bilgilerin sunulacağını anlar. Ancak bu yöntem hem web sayfalarını yavaşlatıyor hem de verileri şifrelemediği için güvenlik zaafları oluşturuyordu.

Local Storage / Session Storage

Yukarıda çerezlerin oluşturduğu sorunlardan dolayı HTML5 ile birlikte local storage ve session storage ile tanıştık. Bu yeni yöntem ile veriler artık sunucularda küçük paketler olarak değil kullanıcının tarayıcısında çok daha büyük depolama imkanıyla saklanıyor. Özünde Local Storage : JavaScript sitelerinin ve uygulamalarının son kullanma tarihi olmadan, bir web tarayıcısında key/value değerlerinin kaydedebilmesine izin veren bir özelliktir. Session Storage'ın tek farkı Local Storage'dan, Açılan sekme kapatıldığında sekmede saklanan veriler de silinir.
